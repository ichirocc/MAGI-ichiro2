# MAGI 勤務表アプリ 画面仕様書

バージョン: 1.0  
対象ブランチ: main  
作成日: 2026-06-16

---

## 1. アプリ概要

**MAGI 勤務表**は、V6 ネイティブ最適化エンジンを搭載したシフトスケジューリング Android アプリである。  
Jetpack Compose で実装されており、SA + ALNS による組み合わせ最適化で複雑な勤務制約を解く。

### 1.1 全体レイアウト構造

```
┌──────────────────────────────────────┐
│  トップバー: "MAGI 勤務表" + ステータスバッジ │
├──────────────────────────────────────┤
│  コンテンツエリア（スクロール可）             │
│  ・StatusHero（常時表示）                │
│  ・タブ別カード群                        │
├──────────────────────────────────────┤
│  ボトムコマンドバー（元に戻す / 主要操作）    │
├──────────────────────────────────────┤
│  ボトムナビ（5タブ）                      │
└──────────────────────────────────────┘
```

### 1.2 ナビゲーション

| インデックス | タブ名 | アイコン | 説明 |
|---|---|---|---|
| 0 | ホーム | Home | ステータス概要・Copilot |
| 1 | 勤務表 | DateRange | スケジュールグリッド・編集 |
| 2 | 編集 | Edit | 問題定義・制約設定 |
| 3 | 分析 | Assessment | 違反内訳・V6 ダッシュボード |
| 4 | 設定 | Settings | テーマ・データ入出力・最適化設定 |

### 1.3 トップバー

| 要素 | 内容 |
|---|---|
| タイトル | "MAGI 勤務表" |
| ステータスバッジ | 以下の4状態 |

**ステータスバッジ状態:**

| 状態 | ラベル | 背景色 | 条件 |
|---|---|---|---|
| 実行中 | "実行中" | primaryContainer | `ui.running == true` |
| 配布可 | "配布可" | tertiaryContainer (緑) | `bestHard == 0` |
| 未解決 | "未解決 N" | errorContainer (赤) | `bestHard > 0` |
| 未計算 | "未計算" | surfaceVariant (灰) | `!hasResult` |

### 1.4 ボトムコマンドバー

| 条件 | 左ボタン | 右ボタン |
|---|---|---|
| `running == true` | 非表示 | "計算を止める" (error色) |
| `!hasResult && !running` | "元に戻す" (canUndo 時) | "勤務表を作成" (primary色) |
| `hasResult && !running` | "元に戻す" (canUndo 時) | "最適化する" (primary色) |

## 2. ホームタブ (Tab 0)

### 2.1 StatusHero カード

最上部に常時表示される勤務表の現況サマリー。

| 状態 | コンテナ色 | アイコン | メッセージ |
|---|---|---|---|
| 未作成 | secondaryContainer | play_circle | "まだ作成していません" + 「右下の『勤務表を作成』から」 |
| 配布可 (HARD=0) | tertiaryContainer | check_circle | "配布できます" + "残り調整 N 件" |
| 未解決 (HARD>0) | errorContainer | warning | "未解決の条件が N 件" + 再試行か設定見直しを促すヒント |

V6 充足分析が有効な場合は充足率・Guard スコアを追加表示。

### 2.2 CopilotCard

最適化の状況を可視化するコーチングカード。

| 要素 | 内容 |
|---|---|
| 満足度スコア | 0〜100（LinearProgressIndicator） |
| 実行中表示 | 残り予算・未解決数・iter/s・現在イテレーション数 |
| 担当外希望警告 | 担当外のシフト希望がある場合に警告＋希望編集リンク |
| ガチャヒント | "同じ設定での再実行で別結果が得られます" |
| 仕上げ消尽通知 | 仕上げ最適化が飽和した場合に通知 |

### 2.3 SummaryCard

| 要素 | 内容 |
|---|---|
| 大統計 3 項目 | HARD 違反数 / SOFT 違反数 / 合計違反数（大フォント） |
| 問題サイズ | スタッフ数・日数・シフト種類数・グループ数 |
| P2 パターン | ON/OFF 表示（MIN=OR ロジック） |
| 初期スコア | 初回 HARD / 初回 SOFT 件数 |

### 2.4 ActionCard

最適化実行ボタン群。

| ボタン名 | 動作 |
|---|---|
| 簡易作成 | `generateSimple()` |
| 最適化 | `runV6FullOptimize()` |
| 高速計算 | `start()` (SA) |
| 軽量 | `runLightOptimize()` |
| バックグラウンド | WorkManager 経由で通知付き最適化 |

実行中は全ボタン無効化。進捗をイテレーション数 / iter/s / 経過時間で表示。

### 2.5 V6DashboardCard

| 要素 | 内容 |
|---|---|
| HARD Core | カバレッジコア評価値 |
| Guard | ガード評価値 |
| 充足 % | カバレッジ充足率（%） |
| リスク上位 | 最リスク高日のラベル・不足数（赤/緑） |
| 日別リスクチップ | スクロール可能なチップ一覧（日付・不足数・詳細） |
| スタッフ負荷 | 上位 5 名の名前・グループ記号・違反数・勤務数・負荷テキスト |

### 2.6 AlternativesCard

並列探索で見つかった代替案リスト。

- 各行: "案N: 必須=X 合計=Y"
- "採用" ボタンで当該案を現在の勤務表として適用

## 7. データモデル

### 7.1 MagiState

| フィールド | 型 | 説明 |
|---|---|---|
| startDate / endDate | String | スケジュール期間 |
| shifts | List\<Shift\> | シフト定義 |
| groups | List\<Group\> | グループ定義 |
| staff | List\<Staff\> | スタッフ定義 |
| schedule | Array\<IntArray\> | schedule[i][j] = シフトインデックス（-1=未割当） |
| wishes | Map\<String, Int\> | "i,j" → シフトインデックス |
| staffRange | Map\<String, Range\> | "i,k" → 下限・上限 |
| needDay1/2 | Map\<String, Int\> | "k,j" → P1/P2 必要人数上書き |
| groupShift | Array\<BooleanArray\> | [g][k] 担当可能マトリクス |
| cons1〜42 | List\<...Cons\> | 各制約ファミリーのリスト |
| shiftColors | Map\<String, String\> | 記号 → "#rrggbb" |
| use2Patterns | Boolean | P2 パターン有効フラグ |
| extras | Map\<String, String\> | 拡張データ |

### 7.2 データ変更時の連鎖処理

```
ユーザー編集
  → MagiState 更新
    → structureEdited = true
    → refreshCheck() （即時再チェック）
    → autoSave() （1200ms デバウンス → filesDir/magi_autosave.json）
```

### 7.3 Undo/Redo

- スタック深度: 最大 30 スナップショット
- スナップショット内容: MagiState + schedule 2D 配列
- クリアタイミング: ファイル読込時
- プッシュタイミング: セル編集・制約変更・最適化開始
